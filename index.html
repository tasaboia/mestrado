Text Classification — Classificação de Texto

▼
Intuição: Dado um documento (texto), queremos dizer automaticamente a que categoria pertence. Ex: spam/não-spam, positivo/negativo, tema do artigo.

Definição Formal

Dado um espaço de documentos X e um conjunto de classes C = {c₁, c₂, ..., cₙ}, uma função de classificação é: f: X → C

O objetivo é aprender esta função a partir de exemplos (treino) e aplicá-la a novos documentos.

Pipeline de Aprendizagem Supervisionada

Training Set (Treino)

Conjunto de documentos com labels conhecidas. O modelo aprende padrões a partir destes exemplos. Quanto maior e mais diverso, melhor a generalização.

Test Set (Teste)

Conjunto separado de documentos não vistos durante o treino. Serve para avaliar a performance real do modelo. Nunca use o test set para tomar decisões de modelação!

Overfitting vs Underfitting

Overfitting (Sobre-ajuste)

O modelo "decorou" o treino mas não generaliza. Alta accuracy no treino, baixa no teste.

Causas: Modelo muito complexo, poucos dados, sem regularização

Sinal: Train acc ≈ 99%, Test acc ≈ 60%

Underfitting (Sub-ajuste)

O modelo é simples demais para capturar padrões. Baixa accuracy tanto no treino como no teste.

Causas: Modelo muito simples, features insuficientes

Sinal: Train acc ≈ 60%, Test acc ≈ 58%

Algoritmos de Classificação

AlgoritmoIdeia BaseVantagensDesvantagensQuando Usar
Naive BayesP(classe|doc) ∝ P(doc|classe)·P(classe)Rápido, funciona com poucos dados, baseline forteAssume independência entre palavras (ingénuo)Classificação de emails, primeiro modelo a testar
Logistic RegressionAprende pesos para features, sigmoid para probabilidadeInterpretável, probabilístico, regularização fácilLinear, precisa de mais dadosBaseline sólido, features esparsas como TF-IDF
SVMHiperplano que maximiza margem entre classesFunciona bem em espaços de alta dimensão, robustoLento em grandes datasets, difícil de interpretarTexto (alta dimensionalidade), datasets médios

Como Naive Bayes Funciona (Detalhe)

Usando o Teorema de Bayes: P(c|d) = P(d|c) · P(c) / P(d)

Assumindo independência condicional das palavras (assumption "ingénua"):

P(c|d) ∝ P(c) · ∏ P(wᵢ|c) para cada palavra wᵢ no documento d

Na prática usa-se log-probabilidades para evitar underflow numérico:

log P(c|d) ∝ log P(c) + Σ log P(wᵢ|c)

Laplace Smoothing: Para palavras não vistas no treino, usa-se P(w|c) = (count(w,c) + 1) / (count(c) + |V|) onde |V| é o tamanho do vocabulário.

Erro Comum no Exame

❌ "Naive Bayes é melhor que SVM porque considera probabilidades"

✅ A qualidade depende do dataset. A "ingenuidade" da independência de palavras é uma limitação forte — "não gosto" quebra essa independência. SVM geralmente supera NB em textos longos.

Text Preprocessing — Pré-processamento

▼
Intuição: Texto bruto é "sujo". Antes de qualquer análise, precisamos normalizar e estruturar o texto. Garbage in → Garbage out.
TécnicaO Que FazExemploNotas
TokenizationDivide texto em unidades (tokens)"Bom dia!" → ["Bom", "dia", "!"]Pode ser por palavra, subpalavra, caractere
Sentence SegmentationIdentifica fronteiras de frases"Dr. Smith chegou. Ele sorriu." → 2 frasesDifícil com abreviações ("Dr.", "etc.")

Stemming vs Lemmatization — Diferença Crucial

Stemming (Porter Stemmer)

"studies"  → "studi"
"studying" → "studi"
"studied"  → "studi"
// Regras heurísticas, não dict
// Resultado pode não ser palavra real

Lemmatization

"studies"  → "study"
"studying" → "study"
"better"   → "good"
// Usa dicionário morfológico
// Resultado é sempre palavra real

Feature Extraction — Bag of Words, N-Grams, TF-IDF

▼
Intuição: Máquinas não entendem texto. Precisamos converter palavras em números. TF-IDF é a técnica clássica mais importante para exame.

Bag of Words (BoW)

Representa um documento como um vetor de contagens de palavras. Ignora a ordem. Exemplo:

Doc1: "o gato comeu o rato"

Doc2: "o rato fugiu"

ogatocomeuratofugiu
Doc121110
Doc210011

N-Grams

Sequências de N palavras consecutivas. Capturam contexto parcial:

"text mining is fun"
Unigrams  (N=1): ["text", "mining", "is", "fun"]
Bigrams   (N=2): ["text mining", "mining is", "is fun"]
Trigrams  (N=3): ["text mining is", "mining is fun"]

Vantagem: Capta frases como "not good" (bigrama) vs apenas "not" e "good" separados.

TF-IDF — Cálculo Passo a Passo (MUITO IMPORTANTE)

⚠️ Isto SEMPRE cai no exame! Saiba calcular TF-IDF manualmente.

TF (Term Frequency): Frequência do termo no documento

TF(t, d) = count(t in d) / total_words(d)

IDF (Inverse Document Frequency): Penaliza termos que aparecem em muitos documentos

IDF(t, D) = log( N / df(t) )   onde N = nº docs, df(t) = nº docs com o termo t

TF-IDF:

TF-IDF(t, d, D) = TF(t, d) × IDF(t, D)

Exemplo Completo de Cálculo

3 documentos:

D1: "o gato comeu o rato" (5 palavras)

D2: "o rato fugiu rápido" (4 palavras)

D3: "o gato é rápido" (4 palavras)

Calcular TF-IDF da palavra "gato":

PassoCálculoResultado
TF(gato, D1)1 / 50.20
TF(gato, D2)0 / 40.00
TF(gato, D3)1 / 40.25
df(gato)aparece em D1 e D3 = 2 documentos2
IDF(gato)log(3 / 2) = log(1.5)≈ 0.176
TF-IDF(gato, D1)0.20 × 0.176≈ 0.035
TF-IDF(gato, D2)0.00 × 0.176= 0.000
TF-IDF(gato, D3)0.25 × 0.176≈ 0.044

Calcular TF-IDF da palavra "o" (stopword típica):

PassoCálculoResultado
df("o")aparece em D1, D2 e D3 = 3 docs3
IDF("o")log(3 / 3) = log(1)= 0
TF-IDF("o", qualquer doc)TF × 0= 0
Insight crucial: TF-IDF automaticamente dá peso 0 a palavras que aparecem em todos os documentos (como stopwords). Palavras raras e específicas recebem peso alto.

Cosine Similarity — Medida de Similaridade

cos(A, B) = (A · B) / (||A|| × ||B||)

Mede o ângulo entre dois vetores TF-IDF. Valor entre 0 (totalmente diferentes) e 1 (idênticos).

Porquê cosine e não distância euclidiana? A distância euclidiana é afetada pelo comprimento do documento. Documentos longos teriam sempre maior distância. Cosine normaliza pelo comprimento, sendo mais justo.

Sentiment Analysis — Análise de Sentimento

▼
Intuição: Extrair opiniões, sentimentos e atitudes de texto. "Este produto é fantástico!" → Positivo. "Péssimo serviço." → Negativo.

Hierarquia da Análise de Sentimento

NívelDefiniçãoExemplo
Opinion MiningExtrair opiniões sobre entidades/aspectos"A câmara do iPhone é boa, mas a bateria é fraca"
Subjectivity AnalysisDistinguir texto subjetivo de objetivoObjetivo: "O iPhone pesa 174g" / Subjetivo: "O iPhone é elegante"
Sentiment MiningPolarity + intensidadePositivo (0.8), Negativo (0.2), Neutro (0.0)

Granularidade de Análise

Documento

Análise de toda a review/texto. Ex: "Esta review é positiva?"

Frase

Cada frase separada. Permite sentimentos mistos no mesmo documento.

Aspeto (ABSA)

Aspeto-Based Sentiment Analysis: "câmara:positivo, bateria:negativo"

Sentiment Lexicons

General Inquirer

Dicionário com 11,788 palavras marcadas como positive/negative/neutral. Desenvolvido em Harvard. Abordagem manual — linguistas categorizaram cada palavra.

Limitação: Não captura contexto. "banco" pode ser financeiro ou de jardim.

LIWC (Linguistic Inquiry and Word Count)

Analisa dimensões psicológicas e linguísticas: emoções, cognição, processos sociais. Conta % de palavras em cada categoria.

Limitação: Foco em psicologia, não sempre adequado para reviews comerciais.

Negation Handling — Tratamento de Negação

Problema crítico: "Este filme é bom" vs "Este filme não é bom". Sem tratamento, ambos ativariam o token positivo "bom"!

Estratégia NOT-prefix: Após um "não/not", prefixar as próximas N palavras com "NOT_":

"não gosto disto" → "não NOT_gosto NOT_disto"

Problemas linguísticos:

  • Scope ambíguo: "Não gosto de filmes de ação ou comédia" — negação aplica a ambos?
  • Dupla negação: "Não é mau" → Positivo!
  • Negação longa-distância: "O que eu não esperava era gostar tanto"
  • Ironias e sarcasmo: "Claro que adorei esperar 2 horas" → Negativo

Aplicações por Domínio

DomínioAplicaçãoDesafio Específico

Netflix/CinemaQualidade de conteúdo percebidaGíria, referências culturais

Topic Modelling — LDA

▼
Intuição: Sem labels. Queremos descobrir automaticamente os "temas" escondidos numa coleção de documentos. Unsupervised.

LDA — Latent Dirichlet Allocation

Pressupostos do modelo:

  • Cada documento é uma mistura de tópicos: "40% política, 30% economia, 30% desporto"
  • Cada tópico é uma distribuição sobre palavras: tópico "desporto" tem alta prob. para "gol", "jogo", "equipa"
  • Cada palavra no documento foi gerada: escolher tópico → escolher palavra desse tópico

Intuição Matemática do LDA

O modelo generativo assume que os documentos foram criados assim:

1
Para cada documento d, amostrar distribuição de tópicos θ_d ~ Dirichlet(α)
2
Para cada tópico k, amostrar distribuição de palavras β_k ~ Dirichlet(η)
3
Para cada palavra na posição n do documento: escolher tópico z_n ~ Multinomial(θ_d); gerar palavra w_n ~ Multinomial(β_{z_n})

O que LDA estima: θ (distribuição de tópicos por documento) e β (distribuição de palavras por tópico) a partir das palavras observadas.

Parâmetros: K (número de tópicos — você escolhe), α (concentração prior dos tópicos), η (concentração prior das palavras)

Word Embeddings — Word2Vec, GloVe, FastText

▼
Intuição: TF-IDF trata palavras como símbolos independentes. Word embeddings representam palavras como vetores densos onde palavras similares ficam próximas no espaço.

Representação Vetorial

Cada palavra é mapeada para um vetor de dimensão d (tipicamente 100-300). Palavras semanticamente similares têm vetores similares.

rei → [0.7, 0.2, -0.1, ..., 0.4] rainha → [0.6, 0.3, -0.1, ..., 0.5] homem → [0.5, -0.1, 0.8, ..., 0.1] mulher → [0.4, -0.1, 0.7, ..., 0.2]

Word2Vec — Dois Modelos

CBOW (Continuous Bag of Words)

Contexto → Palavra central

"O ___ salta sobre a lua" → prediz "gato"

Janela de contexto (ex: 5 palavras à volta)

Treina mais rápido, melhor para vocabulários frequentes

Skip-gram

Palavra central → Contexto

"gato" → prediz ["O", "salta", "sobre"]

Aprende relações mais ricas

Melhor para palavras raras e grandes corpora

O Famoso Exemplo: Rei - Homem + Mulher = Rainha

vec("rei") - vec("homem") + vec("mulher") ≈ vec("rainha")

Porquê isto acontece?

Word2Vec aprende a partir de padrões de co-ocorrência. "Rei" e "rainha" aparecem em contextos similares (palácio, coroa, reino). "Homem" e "mulher" aparecem em contextos genéricos com o mesmo padrão de diferença de género.

O vetor vec("rei") - vec("homem") captura a dimensão "realeza". Adicionando vec("mulher") desloca para o equivalente feminino, que se aproxima de vec("rainha").

Outros exemplos: Paris - França + Itália ≈ Roma; médico - homem + mulher ≈ enfermeira

Comparação GloVe vs Word2Vec vs FastText

ModeloAbordagemVantagensDesvantagem
Word2VecPredição local (janela)Rápido, analogias, semânticaOOV problem, sem morfologia
GloVeFatorização de co-ocorrência globalCaptura estatísticas globais do corpus, bom em analogiasOOV problem, mais lento
FastTextSubwords (n-gramas de caracteres)Lida com OOV e palavras morfologicamente ricas. Excelente para Português!Vetores maiores

OOV = Out Of Vocabulary: Palavras não vistas durante o treino. FastText resolve isto usando subwords — mesmo sem ver "incompreensibilidade", consegue estimá-la via "compreen", "sivel", etc.

Transformers & LLMs

▼
Paper fundamental: "Attention Is All You Need" (Vaswani et al., 2017). Revolucionou NLP ao eliminar RNNs e usar apenas mecanismos de atenção.

Motivação

RNNs processam sequências palavra-a-palavra, criando "gargalo" de informação. Long-range dependencies são difíceis. Transformers processam toda a sequência em paralelo.

Arquitetura Transformer

Encoder

  • Processa a sequência de input
  • Produz representação contextualizada
  • Cada palavra "vê" toda a frase
  • Usado em: BERT, classificação, tradução (source)

Decoder

  • Gera output token por token
  • Masked self-attention (não vê o futuro)
  • Cross-attention com encoder
  • Usado em: GPT, geração de texto

Self-Attention — O Coração do Transformer

Para cada palavra, calcula três vetores: Query (Q), Key (K), Value (V)

Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V

Intuição:

  • Q: "O que esta palavra está a procurar?"
  • K: "O que esta palavra oferece para outros?"
  • V: "Que informação esta palavra contribui?"
  • √d_k: Fator de escala para estabilizar gradientes

Exemplo: Em "O banco do jardim estava molhado", a palavra "banco" usa atenção para perceber o contexto: alta atenção a "jardim" → banco=assento, não banco financeiro.

Multi-Head Attention

Em vez de uma atenção, usa-se h atenções em paralelo. Cada "cabeça" aprende um tipo diferente de relação (sintáctica, semântica, co-referência...)

MultiHead(Q,K,V) = Concat(head₁, ..., headₕ) · Wᴼ

Large Language Models

ModeloOrganizaçãoArquiteturaInovação
GPTOpenAIDecoder-onlyPre-training + fine-tuning; RLHF; escala
ClaudeAnthropicDecoder-onlyConstitutional AI, alinhamento, safety
GeminiGoogleMultimodal TransformerNativo multimodal (texto+imagem+áudio)
BERTGoogleEncoder-onlyBidirectional, masked language modeling

Como derivam dos Transformers: LLMs são Transformers treinados em enormes quantidades de texto. Emergem capacidades como raciocínio, few-shot learning, e seguir instruções — sem serem explicitamente programadas.

Componentes de uma Série Temporal

▼
Intuição: Uma série temporal é a sobreposição de vários padrões. Decompor = separar esses padrões para analisá-los individualmente.

As 4 Componentes

ComponenteDefiniçãoExemplo RealComo Detectar
Tendência (Trend)Movimento de longo prazo para cima ou baixo (ou estável)Crescimento das vendas de EVs 2015-2025Média móvel de longo prazo, regressão linear
SazonalidadePadrão periódico fixo e conhecido (mensal, semanal, anual)Vendas de sorvete altas no verão; vendas de natal em dezembroACF com picos em múltiplos do período

Modelo Aditivo vs Multiplicativo

Aditivo: Y = T + S + C + ε

A variância sazonal é constante independentemente do nível da série.

Quando usar: Série com amplitude sazonal constante ao longo do tempo

Multiplicativo: Y = T × S × C × ε

A variância sazonal cresce com o nível da série.

Quando usar: Série cujas flutuações aumentam quando os valores crescem (mais comum em dados económicos)

2

Estacionaridade — Conceito Fundamental

▼
Intuição: Uma série é estacionária se as suas propriedades estatísticas não mudam ao longo do tempo. É como um jogo com regras fixas — você pode aprender essas regras e fazer previsões.

Definição Formal — Estacionaridade Fraca

Uma série Yₜ é fracamente estacionária se:

  1. Média constante: E[Yₜ] = μ para todo t
  2. Variância constante (finita): Var[Yₜ] = σ² para todo t
  3. Autocovariância depende só do lag: Cov(Yₜ, Yₜ₊ₖ) = γ(k) — depende só de k, não de t

Porquê a Estacionaridade é Crucial?

Os modelos ARIMA assumem estacionaridade. Numa série não-estacionária, os parâmetros mudam ao longo do tempo — o modelo do passado não serve para o futuro.

Tipos de Não-Estacionaridade e Soluções

TipoCaracterísticaSolução
Tendência determinísticaMédia cresce/decresce sistematicamenteRemover tendência (detrending) ou diferenciar
Random WalkYₜ = Yₜ₋₁ + εₜ (unit root)1ª diferença: ΔYₜ = Yₜ - Yₜ₋₁
SazonalidadePadrão periódico na médiaDiferença sazonal: ΔₛYₜ = Yₜ - Yₜ₋ₛ
Variância crescenteHeterocedasticidadeTransformação log ou Box-Cox

Teste ADF (Augmented Dickey-Fuller)

H₀: A série tem raiz unitária (não-estacionária)

H₁: A série é estacionária

Se p-value < 0.05: Rejeitar H₀ → série é estacionária

Se p-value ≥ 0.05: Não rejeitar H₀ → série é não-estacionária → diferenciar e testar de novo

3

ACF e PACF — Identificação do Modelo

▼
Intuição: ACF e PACF são gráficos de diagnóstico que nos dizem que tipo de modelo (AR ou MA) e quais os parâmetros usar. São o coração do método Box-Jenkins.

ACF (Autocorrelation Function)

Correlação entre Yₜ e Yₜ₋ₖ para cada lag k. Inclui efeitos indiretos (através de lags intermédios).

ρ(k) = Cor(Yₜ, Yₜ₋ₖ) = γ(k) / γ(0)

PACF (Partial ACF)

Correlação entre Yₜ e Yₜ₋ₖ após remover o efeito dos lags intermédios (lag 1, 2, ..., k-1). Mede o efeito "direto".

Guia de Identificação ARIMA(p,d,q)

ModeloACFPACF
AR(p)Decai gradualmente (exponencial ou oscilatório)Corta bruscamente após lag p
MA(q)Corta bruscamente após lag qDecai gradualmente
ARMA(p,q)Decai gradualmente após lag qDecai gradualmente após lag p
White NoiseTodos os lags ≈ 0 (dentro das bandas)Todos os lags ≈ 0
Regra prática: Bandas de confiança a ±1.96/√n. Picos fora das bandas são estatisticamente significativos. Se ACF decai e PACF corta em lag 2 → modelo AR(2).
4

AR, MA, ARMA, ARIMA, SARIMA

▼

AR(p) — AutoRegressive

Yₜ é regressão linear nos seus próprios valores passados:

Yₜ = c + φ₁Yₜ₋₁ + φ₂Yₜ₋₂ + ... + φₚYₜ₋ₚ + εₜ

"Hoje depende dos últimos p dias." Ex: AR(1) — preço de amanhã ≈ φ₁ × preço de hoje + ruído

MA(q) — Moving Average

Yₜ é combinação linear dos erros passados:

Yₜ = μ + εₜ + θ₁εₜ₋₁ + θ₂εₜ₋₂ + ... + θᵩεₜ₋ᵩ

"Hoje é afetado por choques/erros dos últimos q períodos." MA é sempre estacionário.

ARMA(p,q)

Combinação: Yₜ = c + Σφᵢ Yₜ₋ᵢ + εₜ + Σθⱼ εₜ₋ⱼ

ARIMA(p, d, q) — Com Integração

Para séries não-estacionárias. Diferencia d vezes antes de aplicar ARMA:

ParâmetroSignificadoComo Determinar
pOrdem AR — quantos lags de YPACF: onde corta bruscamente
dGrau de diferenciação para estacionarizarTeste ADF: quantas diferenças até estacionarizar
qOrdem MA — quantos lags de erroACF: onde corta bruscamente
Critérios de Seleção: AIC (Akaike) e BIC (Bayesian). Menor é melhor. BIC penaliza mais a complexidade → modelos mais parsimoniosos.

SARIMA(p,d,q)(P,D,Q)[s] — Com Sazonalidade

Extensão do ARIMA para séries com sazonalidade de período s:

  • (p,d,q) — componente não-sazonal (como ARIMA normal)
  • (P,D,Q) — componente sazonal (AR, diferenciação, MA sazonais)
  • [s] — período sazonal: s=12 para mensal, s=4 para trimestral, s=7 para diário

Exemplo: SARIMA(1,1,1)(1,1,1)[12] para dados mensais com sazonalidade anual.

Holt-Winters — Suavização Exponencial Tripla

▼
Intuição: Atualiza 3 componentes dinamicamente: nível (L), tendência (B), sazonalidade (S). Observações recentes têm mais peso (exponential decay).

Equações de Holt-Winters Aditivo

Lₜ = α(Yₜ - Sₜ₋ₛ) + (1-α)(Lₜ₋₁ + Bₜ₋₁)   [Nível] Bₜ = β(Lₜ - Lₜ₋₁) + (1-β)Bₜ₋₁   [Tendência] Sₜ = γ(Yₜ - Lₜ) + (1-γ)Sₜ₋ₛ   [Sazonalidade] Ŷₜ₊h = Lₜ + h·Bₜ + Sₜ₊ₕ₋ₛ   [Previsão h passos à frente]

α, β, γ ∈ [0,1] — parâmetros de suavização. Valores próximos de 1 dão mais peso à observação atual. Otimizados por mínimos quadrados.

1

Probabilidade Condicional e Teorema de Bayes

▼
Intuição: Bayes é uma fórmula para atualizar crenças com nova evidência. Começa com o que você acredita (prior), vê dados, e atualiza (posterior).

Probabilidade Condicional

P(A|B) = P(A ∩ B) / P(B)

Probabilidade de A dado que B já ocorreu. Restringe o espaço amostral.

Derivação do Teorema de Bayes

Da definição de probabilidade condicional:

P(A|B) = P(A ∩ B) / P(B)    e    P(B|A) = P(A ∩ B) / P(A)

Logo: P(A ∩ B) = P(A|B)·P(B) = P(B|A)·P(A)

P(A|B) = P(B|A) · P(A) / P(B)

Teorema de Bayes para Inferência Estatística

P(θ | data) = P(data | θ) · P(θ) / P(data)
TermoNomeSignificado
P(θ | data)PosteriorO que acreditamos sobre θ APÓS ver os dados
P(data | θ)LikelihoodProbabilidade dos dados dado que θ é o verdadeiro valor
P(θ)PriorO que acreditamos sobre θ ANTES de ver os dados
P(data)Evidence/Normalizing constantGarante que posterior soma 1. Muitas vezes ignorado: "∝"
Posterior ∝ Likelihood × Prior

Exemplo Concreto: Teste de Doença

Doença tem prevalência 1% (P(D) = 0.01). Teste com sensibilidade 99% (P(+|D) = 0.99) e especificidade 95% (P(-|¬D) = 0.95, logo P(+|¬D) = 0.05).

Qual a probabilidade de ter a doença dado teste positivo?

P(D|+) = P(+|D)·P(D) / P(+) P(+) = P(+|D)·P(D) + P(+|¬D)·P(¬D) = 0.99×0.01 + 0.05×0.99 = 0.0099 + 0.0495 = 0.0594 P(D|+) = (0.99 × 0.01) / 0.0594 ≈ 0.167 = 16.7%
Insight: Mesmo com teste 99% preciso, se a doença é rara (1%), um teste positivo significa apenas 16.7% de chance de ter a doença! Isto é o paradoxo do teste de rastreio — contraintuitivo mas matematicamente correto.
2

Prior, Likelihood, Posterior — MAP vs MLE

▼

O Papel do Prior

Prior Informativo

Incorpora conhecimento real. Ex: sabemos que uma moeda é "provavelmente justa", então usamos Beta(10,10) — concentrado em 0.5.

Vantagem: Com poucos dados, o prior guia para resultados razoáveis

Risco: Prior errado pode enviesar resultados

Prior Não-Informativo (Flat/Diffuse)

Ex: Beta(1,1) = Uniforme. "Não sei nada." Todos os valores igualmente prováveis.

Resultado: Com prior uniforme, MAP = MLE

Jeffreys Prior: Prior não-informativo invariante a reparametrizações

MLE vs MAP

MLE — Maximum Likelihood Estimation

Encontra θ que maximiza P(data|θ). Ignora prior.

θ̂_MLE = argmax P(data | θ)

Frequentista. Trata θ como constante desconhecida.

Com n→∞, MLE é ótimo. Com poucos dados, pode overfit.

MAP — Maximum A Posteriori

Encontra θ que maximiza P(θ|data) ∝ P(data|θ)·P(θ).

θ̂_MAP = argmax P(data|θ)·P(θ)

Bayesiano. O prior atua como regularizador!

MAP com prior gaussiano = Ridge Regression (L2)

MAP com prior Laplace = Lasso (L1)

Distribuições Conjugadas — Crucial para Exame

Definição: Uma família de priors é conjugada ao likelihood se o posterior pertence à mesma família do prior. Permite cálculo analítico!
PriorLikelihoodPosteriorAplicação
Beta(α,β)Binomial(n,θ)Beta(α+k, β+n-k)Proporções, taxas de sucesso
Normal(μ₀,σ₀²)Normal(μ,σ²) com σ² conhecidoNormal(μₙ, σₙ²)Modelos lineares, média populacional
Gamma(α,β)Poisson(λ)Gamma(α+Σy, β+n)Contagens, taxas de eventos
Dirichlet(α)Multinomial(n,θ)Dirichlet(α+counts)Modelos de tópicos, LDA

Exemplo Beta-Binomial Completo

Queremos estimar a probabilidade θ de uma moeda dar cara. Prior: Beta(2,2) (crença de que a moeda é justa). Observamos 7 caras em 10 lançamentos.

Prior: Beta(α=2, β=2) → E[θ] = α/(α+β) = 2/4 = 0.5 Likelihood: Binomial(n=10, k=7) Posterior: Beta(α+k, β+n-k) = Beta(2+7, 2+3) = Beta(9, 5) E[θ|data] = 9/(9+5) = 9/14 ≈ 0.643

Interpretação: Começamos a achar que a moeda é justa (0.5). Com 7/10 caras, atualizamos para 0.643. O prior "puxou" a estimativa para baixo relativamente ao MLE (0.7).

Credible Intervals (Intervalos de Credibilidade)

Um intervalo de 95% credible: "Há 95% de probabilidade de θ estar neste intervalo." Interpretação direta e intuitiva.

Contraste com Frequentista: Um IC 95% frequentista NÃO significa "95% de prob. de θ estar aqui" — significa "95% dos ICs construídos deste modo conteriam o θ verdadeiro". Isto é muito mais difícil de comunicar!
3

MCMC — Markov Chain Monte Carlo

▼
Intuição: Quando o posterior não tem forma analítica conhecida, precisamos de amostras. MCMC é um algoritmo para amostrar de distribuições complexas sem precisar de calculá-las explicitamente.

Porquê MCMC?

Em modelos complexos com muitos parâmetros, P(data) = ∫P(data|θ)P(θ)dθ é intratável. MCMC contorna este problema: em vez de calcular o posterior, amostras dele.

Metropolis-Hastings Algorithm

1
Inicializar θ₀ (qualquer valor razoável)
2
Propor θ* ~ q(θ*|θₜ) (distribuição de proposta, ex: Normal centrada em θₜ)
3
Calcular α = min(1, [P(data|θ*)P(θ*)] / [P(data|θₜ)P(θₜ)]) — ratio de posteriors
4
Aceitar θ* com probabilidade α. Se aceite, θₜ₊₁ = θ*. Senão, θₜ₊₁ = θₜ
5
Repetir. Após burn-in, as amostras convergem para o posterior.

🔴 Text Mining — Questões Curtas

TM-Q1 | Resposta Curta
Explique a diferença entre Stemming e Lemmatization. Dê um exemplo de quando Lemmatization seria preferível.
Stemming aplica regras heurísticas para reduzir palavras à sua raiz (pode gerar não-palavras: "studies" → "studi"). Lemmatization usa dicionários morfológicos para obter a forma canônica ("studies" → "study", "better" → "good"). Lemmatization é preferível quando a interpretabilidade importa (e.g., análise linguística) ou quando palavras de classes gramaticais diferentes são relevantes ("is/am/are" → "be" vs Stemming que as trata diferentemente).
TM-Q2 | Cálculo
Calcule TF-IDF da palavra "data" nos documentos D1="data mining is fun", D2="data science uses data", D3="fun with science". Mostre todos os passos. Use log base 10.
TF("data",D1) = 1/4 = 0.25; TF("data",D2) = 2/4 = 0.5; TF("data",D3) = 0/3 = 0
df("data") = 2 docs (D1 e D2)
IDF("data") = log₁₀(3/2) = log₁₀(1.5) ≈ 0.176
TF-IDF(D1) = 0.25 × 0.176 = 0.044
TF-IDF(D2) = 0.5 × 0.176 = 0.088

TM-Q3 | Conceptual
Porquê o TF-IDF atribui peso zero a stopwords como "o", "a", "de"? Explique matematicamente.
Stopwords aparecem em todos (ou quase todos) os documentos. Se uma palavra t aparece em N documentos e o corpus tem N documentos, então IDF(t) = log(N/N) = log(1) = 0. Como TF-IDF = TF × IDF, e IDF = 0, o TF-IDF é 0 independentemente do TF. A intuição é que uma palavra que está em todos os documentos não distingue nenhum → não tem valor discriminativo.
TM-Q4 | Algoritmo
Explique a "ingenuidade" do Naive Bayes e por que, apesar disso, o algoritmo funciona bem em prática para classificação de texto.
A ingenuidade é assumir independência condicional entre as palavras dado a classe: P(w₁,w₂,...|c) = ∏P(wᵢ|c). Isto é claramente falso ("New York" tem dependência forte). Mas NB funciona porque: (1) para classificação o que importa é comparar P(c₁|d)/P(c₂|d) — o numerador/denominador erram na mesma direção; (2) com alta dimensionalidade as violações tendem a se cancelar; (3) é robusto ao overfitting com poucos dados; (4) é extremamente eficiente computacionalmente.
TM-Q5 | Análise
Descreva o problema de Negation Handling em Sentiment Analysis. Dê dois exemplos linguísticos que tornam este problema particularmente difícil.
Negação inverte a polaridade de termos adjacentes: "bom" (positivo) torna-se negativo em "não é bom". A estratégia NOT-prefix trata palavras após negação como opostas. Casos difíceis: (1) Dupla negação — "não é mau" = positivo, mas NOT-prefix marcaria "NOT_mau" como negativo; (2) Scope longo — "O que eu não esperava era gostar tanto" — a negação tem scope complexo que regras simples de janela não capturam; (3) Ironia — "Adorei esperar 2 horas" é negativo apesar de vocabulário positivo.
TM-Q6 | Word Embeddings
Explique geometricamente porquê vec("rei") - vec("homem") + vec("mulher") ≈ vec("rainha").
Word2Vec aprende vetores onde palavras em contextos similares ficam próximas. "Rei" e "rainha" partilham contextos de realeza (coroa, palácio, reino). "Homem" e "mulher" partilham contextos genéricos com diferença apenas de género. O vetor (rei - homem) captura a dimensão "realeza sem género masculino". Adicionando vec("mulher"), deslocamos este vetor para incorporar género feminino, chegando a "rainha" — o resultado combina realeza + feminino. O espaço vetorial captura relações analógicas porque a estrutura distribucional do vocabulário codifica estas relações semânticas.
TM-Q7 | Transformers
Explique o mecanismo de Self-Attention nos Transformers. Para que serve o fator 1/√d_k?
Self-Attention permite que cada posição na sequência "atenda" a todas as outras posições. Para cada palavra são calculados três vetores: Query (o que procuro), Key (o que ofereço) e Value (o que contribuo). O score de atenção entre posição i e j é: score = Qᵢ · Kⱼᵀ / √d_k. Softmax normaliza os scores em pesos. O output é a soma ponderada dos Values. √d_k: sem este fator, com dimensões grandes (d_k=512), os produtos internos ficam muito grandes em magnitude, fazendo o softmax saturar em regiões com gradientes muito pequenos, dificultando o treino.
TM-Q8 | Comparação
Compare Bag-of-Words com Word Embeddings. Em que situações cada abordagem seria preferível?
BoW/TF-IDF: vetores esparsos, alta dimensionalidade (|vocab|), ignora ordem e semântica, sem relação entre palavras similares. Word Embeddings: vetores densos, baixa dimensionalidade (100-300), captura semântica e similaridade, requer grandes corpora de pré-treino. BoW preferível: (1) datasets pequenos sem embeddings pré-treinados; (2) classificação por keywords específicas; (3) interpretabilidade total necessária; (4) línguas com poucos recursos. Embeddings preferíveis: (1) semântica importa (sinónimos devem ser equivalentes); (2) tarefas de similaridade/analogia; (3) modelos downstream deep learning; (4) dados suficientes para treino ou embeddings pré-treinados disponíveis.

🟡 Séries Temporais — Questões

ST-Q1 | Conceptual
O que significa uma série temporal ser "estacionária"? Por que é que esta propriedade é fundamental para a modelação com ARIMA?
Uma série é fracamente estacionária se: (1) média constante E[Yₜ] = μ ∀t; (2) variância constante Var[Yₜ] = σ² ∀t; (3) autocovariância Cov(Yₜ,Yₜ₊ₖ) depende só do lag k, não de t. É fundamental para ARIMA porque: os parâmetros AR/MA assumem relações estáveis ao longo do tempo. Numa série não-estacionária, os padrões mudam — o modelo aprendido com dados passados não generaliza para o futuro. Além disso, testes estatísticos e propriedades teóricas dos estimadores de mínimos quadrados dependem de estacionaridade.
ST-Q2 | Análise
Você observa numa série: ACF decai gradualmente (ondas sinusoidais), PACF corta abruptamente após lag 3. Que modelo sugere? Que parâmetros ARIMA(p,d,q) usaria se a série já for estacionária?
ACF decai gradualmente → processo AR (a ação de AR faz o ACF decair exponencialmente). PACF corta após lag 3 → ordem AR = 3. Logo, sugere modelo AR(3). Em notação ARIMA: ARIMA(3, 0, 0) — p=3 (três lags AR), d=0 (série já estacionária), q=0 (sem componente MA). O modelo seria: Yₜ = c + φ₁Yₜ₋₁ + φ₂Yₜ₋₂ + φ₃Yₜ₋₃ + εₜ
ST-Q3 | Diferença
Explique a diferença entre sazonalidade e ciclo numa série temporal. Por que é que distinguir estas componentes pode ser difícil em prática?
Sazonalidade: padrão periódico com período fixo e conhecido (ex: mensal=12, semanal=7). Ocorre por causas fixas: estações do ano, calendário. Ciclo: flutuações de longo prazo com período variável (geralmente 2-10 anos). Causas: ciclos económicos, tecnológicos. Dificuldades de distinção: (1) o período do ciclo varia — impossível definir s como na sazonalidade; (2) ciclos curtos (2 anos) podem ser confundidos com sazonalidade com s=24 (meses); (3) em séries curtas (menos de 2-3 ciclos), não há dados suficientes para estimar o ciclo; (4) correlação espúria entre tendência e ciclo.
ST-Q4 | Métricas
Compare MAE, RMSE e MAPE. Em que circunstâncias cada métrica seria inapropriada?
MAE = média de |erros| — simples, robusta a outliers, mesma unidade. Inapropriada: quando erros grandes têm custo desproporcional (aviação, medicina). RMSE = raiz de médias de erros² — penaliza erros grandes mais, mesma unidade. Inapropriada: quando há outliers naturais na série que não devem ser penalizados desproporcionalmente. MAPE = erro percentual médio — comparável entre séries de escalas diferentes. Inapropriada: (1) quando Yₜ≈0 (divisão por zero); (2) assimétrica — overforecast e underforecast com mesmo erro absoluto têm MAPE diferente; (3) séries com valores negativos.

🟢 Modelação Bayesiana — Questões

MB-Q1 | Derivação
Derive o Teorema de Bayes a partir da definição de probabilidade condicional. Identifique cada termo na expressão P(θ|data).
Por definição: P(A|B) = P(A∩B)/P(B) → P(A∩B) = P(A|B)·P(B). Simetria: P(A∩B) = P(B|A)·P(A). Logo P(A|B)·P(B) = P(B|A)·P(A) → P(A|B) = P(B|A)·P(A)/P(B). Para inferência: P(θ|data) = P(data|θ)·P(θ)/P(data). Posterior P(θ|data): crença sobre θ após observar dados. Likelihood P(data|θ): prob. dos dados se θ fosse o valor verdadeiro. Prior P(θ): crença inicial sobre θ. Evidence P(data): constante normalizadora = ∫P(data|θ)P(θ)dθ.
MB-Q2 | Cálculo
Prior Beta(3,3). Observa 8 sucessos em 12 tentativas. Calcule: (a) o posterior; (b) a média do posterior; (c) compare com MLE.
(a) Posterior = Beta(α+k, β+n-k) = Beta(3+8, 3+4) = Beta(11, 7). (b) E[θ|data] = α/(α+β) = 11/(11+7) = 11/18 ≈ 0.611. (c) MLE = k/n = 8/12 = 0.667. O prior Beta(3,3) tem média 0.5, "puxando" a estimativa bayesiana de 0.667 para 0.611. Com mais dados, o posterior converge para MLE. Com menos dados, o prior tem mais influência — isto é o "regularizing effect" do prior.
MB-Q3 | Discursiva
Discuta as vantagens e limitações da abordagem bayesiana versus frequentista para análise de dados com amostras pequenas.
Bayesiana com amostras pequenas — Vantagens: (1) Prior incorpora conhecimento domínio, prevenindo estimativas absurdas; (2) Quantifica incerteza parametricamente via distribuição posterior; (3) Credible intervals têm interpretação direta ("95% prob de θ ∈ [a,b]"); (4) Não depende de assintóticos. Limitações: (1) Resultados sensíveis ao prior — escolha subjetiva crítica com poucos dados; (2) Computacionalmente custoso (MCMC); (3) Difícil de comunicar a não especialistas. Frequentista — Vantagens: objetividade (sem prior), familiar, eficiente. Limitações: ICs não têm interpretação direta; com n pequeno, pressupostos assintóticos do MLE falham; sem forma de incorporar conhecimento prévio.
MB-Q4 | MCMC
Explique por que o algoritmo de Metropolis-Hastings funciona: por que amostrar pelo critério de aceitação α converge para amostras do posterior?
Metropolis-Hastings é um algoritmo MCMC que constrói uma cadeia de Markov cuja distribuição estacionária é o posterior P(θ|data). O critério de aceitação α = min(1, P(θ*|data)/P(θₜ|data)) garante detailed balance: a taxa de transição de θₐ para θᵦ é igual à de θᵦ para θₐ ponderada pelos posteriors. Isto implica que a distribuição estacionária é proporcional ao posterior. Na prática: estados com maior posterior são visitados mais frequentemente (pois são mais frequentemente aceites). O ratio do posterior cancela a constante normalizadora P(data) — não precisamos de calculá-la! Com burn-in suficiente, as amostras seguem aproximadamente o posterior.

🔴 Text Mining

O que é TF-IDF e para que serve?
TF-IDF = Term Frequency × Inverse Document Frequency. Pondera palavras sendo: alta freq no doc (TF alto) mas rara no corpus (IDF alto) → peso alto. Serve para representar documentos com destaque para termos discriminativos. Stopwords (em todos os docs) → IDF=0 → TF-IDF=0 automaticamente.
Qual a diferença entre CBOW e Skip-gram no Word2Vec?
CBOW: contexto → palavra central (prediz a palavra central a partir do contexto). Skip-gram: palavra central → contexto (prediz o contexto a partir da palavra central). CBOW: mais rápido, melhor para palavras frequentes. Skip-gram: melhor para palavras raras, captura relações mais ricas.
O que é Self-Attention nos Transformers?
Mecanismo que permite cada posição da sequência "atender" a todas as outras. Calcula Q (Query), K (Key), V (Value) para cada posição. Score = softmax(QKᵀ/√d_k)·V. Output é soma ponderada dos Values pelos scores de atenção. Captura dependências de longa distância sem recorrência.
O que é overfitting e como detectá-lo?
Overfitting: modelo "decora" o treino mas não generaliza. O modelo é complexo demais para os dados disponíveis. Detectar: accuracy/loss muito melhor no treino que no teste (gap grande). Ex: train acc = 98%, test acc = 61%. Soluções: regularização (L1/L2), dropout, mais dados, modelo mais simples, early stopping.
Por que usar Cosine Similarity em vez de Distância Euclidiana para comparar documentos?
Distância Euclidiana é afetada pelo comprimento do documento — um doc longo sempre terá maior distância de outro, mesmo com mesmo tema. Cosine mede o ângulo entre vetores, normalizando pelo comprimento. Dois docs com mesma distribuição de palavras mas um mais longo têm cosine=1 (idênticos) mas distância euclidiana grande. Para texto, o que importa é a proporção de palavras, não o total.

🟡 Séries Temporais

Como usar ACF e PACF para identificar modelo AR(p), MA(q) ou ARMA(p,q)?
AR(p): ACF decai gradualmente (exponencial/oscilatório) + PACF corta bruscamente após lag p → p = último lag significativo no PACF. MA(q): ACF corta bruscamente após lag q + PACF decai gradualmente → q = último lag significativo no ACF. ARMA(p,q): ambas decaem gradualmente. Threshold: picos fora das bandas ±1.96/√n.
O que significam os parâmetros p, d, q no ARIMA(p,d,q)?
p = ordem AR (número de lags de Yₜ usados), determinado pelo PACF. d = grau de diferenciação para estacionarizar (quantas vezes diferenciar até ADF rejeitar H₀). q = ordem MA (número de lags do erro εₜ usados), determinado pelo ACF. ARIMA(1,1,1): 1 lag AR + 1 diferenciação + 1 lag MA.
Qual a diferença entre modelo aditivo e multiplicativo de decomposição?
Aditivo: Y = T + S + C + ε — amplitude sazonal constante ao longo do tempo. Multiplicativo: Y = T × S × C × ε — amplitude sazonal proporcional ao nível da série (cresce com os valores). Detectar: se flutuações sazonais aumentam quando a série sobe → multiplicativo. Transformação log converte multiplicativo em aditivo: log(Y) = log(T) + log(S) + ...
Quando usar MAPE vs RMSE?
MAPE: útil para comparar modelos em séries com escalas diferentes (é percentagem). NÃO usar quando Yₜ≈0 (divisão por zero). RMSE: útil quando erros grandes têm custo desproporcional (penaliza quadraticamente). Mesma unidade da série. NÃO ideal se houver outliers naturais na série que não devem ser penalizados. MAE: robusto, simples, boa opção default.

🟢 Modelação Bayesiana

Enuncie o Teorema de Bayes e identifique cada termo.
P(θ|data) = P(data|θ) · P(θ) / P(data). Posterior = crença sobre θ após ver dados. Likelihood = prob. dos dados dado θ. Prior = crença antes dos dados. Evidence = constante normalizadora. Forma prática: Posterior ∝ Likelihood × Prior (ignorar evidence).
O que são distribuições conjugadas? Dê o exemplo Beta-Binomial.
Prior conjugado: prior e posterior pertencem à mesma família de distribuições. Permite cálculo analítico do posterior. Beta-Binomial: Prior Beta(α,β) + Likelihood Binomial(n,k) → Posterior Beta(α+k, β+n-k). Ex: Prior Beta(2,2), observar 7 caras em 10 → Posterior Beta(9,5). Média posterior = 9/14 ≈ 0.643 vs MLE = 7/10 = 0.70.
Qual a diferença filosófica entre MLE e MAP?
MLE (frequentista): θ̂ = argmax P(data|θ) — encontra θ que torna os dados mais prováveis, sem prior. MAP (bayesiano): θ̂ = argmax P(θ|data) ∝ P(data|θ)·P(θ) — maximiza posterior, o prior atua como regularizador. Com prior uniforme, MAP = MLE. MAP com prior gaussiano = Ridge Regression. MAP com prior Laplace = Lasso. Com n→∞, ambos convergem.
O que é MCMC e porquê é necessário?
MCMC (Markov Chain Monte Carlo): família de algoritmos para amostrar de distribuições complexas. Necessário quando: posterior não tem forma analítica conhecida (modelo complexo, muitos parâmetros). A integral P(data) = ∫P(data|θ)P(θ)dθ é intratável. MCMC constrói cadeia de Markov cuja distribuição estacionária é o posterior — sem precisar calcular a normalizing constant. Metropolis-Hastings: propõe θ*, aceita com prob. α = min(1, ratio de posteriors).
Como interpretar um Credible Interval bayesiano vs Intervalo de Confiança frequentista?
Credible Interval 95%: "Dado os dados observados, há 95% de probabilidade de θ estar neste intervalo." Interpretação direta e intuitiva. IC 95% frequentista: "Se repetíssemos o experimento muitas vezes e construíssemos um IC em cada repetição, 95% dos ICs conteriam o θ verdadeiro." θ é fixo — não é probabilístico. O IC de um único experimento ou contém θ ou não. Esta distinção é fundamental para comunicação científica.
